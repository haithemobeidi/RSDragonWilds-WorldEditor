### 3. 🎯 Phase 2 — .pak extraction (commits `89fe71b`, `11ac701`)

**The goal:** Map the base64 GUIDs in save files (like `AsyEt0xjewNEzYmzr_kg_w`) to real item/quest names.

**Process (~4 hours of investigation):**
1. Installed FModel (Windows GUI .pak browser)
2. Hit "unversioned properties" error → needed `.usmap` mapping file
3. Found the [RSDragonwilds FModel Guide](https://dragonwilds.runescape.wiki/w/User:Keplare/FModel_Guide) on the game wiki
4. Downloaded Keplare's mapping file `5.4.4-178266+++dominion+staging-RSDragonwilds.usmap` from the FModel Discord community channel (`#game-compatibility` → Dragonwilds thread)
5. Configured FModel with `GAME_UE5_LATEST` (the wiki said `GAME_UE5_4` but that gave "read size is bigger than remaining archive length" errors — `GAME_UE5_LATEST` worked)
6. Loaded `RSDragonwilds-Windows.utoc` (NOT the .pak)
7. Bulk-exported `Content/Gameplay/` tree → JSON files via "Export Folder's Packages Properties"
8. User exported 2,921 JSON files to `~\OneDrive\Desktop\Output\Exports\RSDragonwilds\Content\Gameplay\`

**The extractor — `scripts/build_guid_map.py`:**
- Walks every .json recursively
- For each entry, checks Type + Package path to classify (item / quest / mount / skill_perk / attribute / consumable / food / plan / potion / equipment / recipe / npc / unknown)
- Extracts `PersistenceID` (the base64 GUID the save uses), `InternalName`, display name from one of `Name/QuestName/DisplayName/SpellName/...` fields
- Handles 12 different entity type patterns including all `BP_Consumables_*` blueprints
- Outputs to `data/guid_map.json`: `{ "by_persistence_id": { guid: entry }, "by_internal_name": { ... } }`

**Result: 947 GUIDs mapped**
- 208 items
- 142 attributes
- 126 skill perks
- 100 consumable plans
- 89 food items
- 88 wearable equipment
- 66 consumables
- 66 recipe unlockers
- 36 potions

- Plus misc

**Wiring into the parser (parser.py):**
- Module-level `_GUID_MAP` loaded at import from `data/guid_map.json`
- **Manual override layer** (`data/guid_map_overrides.json`) applied on top — for runtime-instance GUIDs that don't match templates
- `lookup_guid(guid)` helper
- `CharacterSave.get_summary()` enriches each quest and inventory item with `display_name`, `description`, `icon_key`

**Wiring into the UI:**
- `tab_quests.html` shows the display_name in gold above the GUID/objective when available
- `tab_inventory.html` shows the item name in gold above the internal ID when available

### 4. Quest dedupe + manual overrides (commit `11ac701`)

**Problem discovered during testing:** user's save had 19 quest entries but only 15 unique IDs — the game accumulates duplicate `QuestProgress.Quests` entries across play sessions. The editor was showing all 19 as separate rows.

**Manual override system:** `data/guid_map_overrides.json` — user-editable JSON mapping any GUID to a catalog entry. Applied on top of the extracted map. **A) Quest variable fingerprinting** — each quest template has a set of QuestBool and QuestInt variables with specific names (e.g. `HasTriggeredEntryVolume`, `CollectedApples`, etc.). Runtime instances keep the same variable names. We could:
1. Extract variable name sets from every QuestData template file
2. For each runtime quest in the user's save, compute its variable name set
3. Match by exact set (or highest overlap)

This should auto-fix most of the unmapped quests.

**B) Crowdsourced manual overrides** — ship a tool in the editor where users click an unmapped quest → "What is this quest?" → a dropdown of known quest names (from the XLSX catalog) → saves to `guid_map_overrides.json`. Over time, a community-contributed override file builds up.

## 🛠️ Phase 2 Tool/Process Reference (for next time)

### FModel setup
- Download: https://fmodel.app
- UE Version: **`GAME_UE5_LATEST`** (NOT `GAME_UE5_4` despite what the wiki says)
- Load: `RSDragonwilds-Windows.utoc` (NOT the .pak)
- Mapping file: from FModel Discord `#game-compatibility` → Dragonwilds thread → Keplare's most recent post
- Output dir: `~\OneDrive\Desktop\Output\Exports` (default varies by install)

### Bulk export command
Right-click a folder in FModel's tree → **Save Folder's Packages Properties**.
Gives you every .uasset in the folder recursively, as JSON.

### Rebuild the GUID map after a new export
```bash
cd "~/Documents/Vibe Projects/RSDragonwildsSaveEditor"
source venv/bin/activate
python scripts/build_guid_map.py
# optional: specify path  python scripts/build_guid_map.py /path/to/exports/Gameplay
```
This rewrites `data/guid_map.json`. Restart the Flask server to pick up changes.

### Add a manual override for a GUID
Edit `data/guid_map_overrides.json`, add an entry like:
```json
"YOUR_GUID_HERE": {
  "type": "quest",
  "name": "The Quest Title",
  "internal_name": "quest_slug",
  "source": "user_reported"
}
```
Reload the Flask server — no code change needed.
