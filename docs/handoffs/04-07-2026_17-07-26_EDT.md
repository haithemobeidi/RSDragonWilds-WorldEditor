### 2. 🎯 Reverse-engineered the .sav binary chunked format
Built `scripts/structure_research/` with three same-world test snapshots and 8 progressively smarter diff scripts. **Maps the entire file format:**

| Chunk tag | Purpose |
|---|---|
| `SAVE` | top-level container |
| `INFO`/`CINF` | file metadata (timestamp, version, world name) |
| `GLOB` | globals |
| `CNIX` | Class Name IndeX (deduplicated class name string table) |
| `PNIX` | Property Name IndeX (deduplicated property name string table) |
| `CDEF`/`CDVE` | Class Definition / Class Default Value Entry |
| `LVLS` | Levels (holds all per-level data including spawn lists) |
| `SPWN` | one record per placed actor (with following `PROP` block) |
| `PROP` | property values for the preceding actor |
| `CORA` | Component Object Reference Array (subchunk inside SPWN) |
| `Pces` | building pieces |
| `SATS` | Spawn ATtribute Set |
| `CLST` | Class List |

Each chunk: 4-byte ASCII tag, 4-byte little-endian uint32 length, then `length` bytes of body.

### 3. 🎯 Identified the placed-actor record layout (Personal Chest, 589 bytes)

| Offset | Size | Field | Stable across saves? | Per-instance? |
|---|---|---|---|---|
| 0x00 | 4 | record version / class table index (`17 00 00 00`) | ✓ | constant per class |
| 0x04 | 16 | instance GUID | ✓ | unique per chest |
| 0x14 | 48 | 6 doubles — actor transform (pos vec3 + rot/scale vec3) | ✓ | unique per chest |
| 0x44 | 24 | class+component table refs | ✓ | constant per class |
| 0x58 | 4 | `CORA` subchunk start | ✓ | constant |
| 0x68 | ~56 | secondary transform (component child) | ✓ | unique per chest |
| 0x07c, 0x084 | 1 each | tick/hash bytes | ✗ (drift per save) | — |
| 0x100+ | … | embedded `PROPF` property block | mostly ✓ | property values |
| 0x1a8 | 1 | per-class instance index (chest #1=01, chest #2=02) | ✓ | unique per chest |

**Critical finding from B vs C diff:** the SAME chest in two different saves of the same world has **only 2 differing bytes** (the tick/hash drift bytes). Position, instance GUID, class refs, transform, secondary transform — all stable. This makes cross-world transplant much simpler than expected — copy the bytes verbatim, optionally regenerate the instance GUID, fix up containing chunk lengths.

### 4. 🎯 Captured TWO chest class signatures

### 6. 🪲 Free root-cause discovery for Bug #001
The chest schema strings in the diff revealed `OwnerCharacterGuid` as a per-actor property. **This is why converting a world from Standard→Custom breaks locked chests / protection totems / privacy:** the world's `WorldSaveSettings/PlayerOwnerGuid` changes, but every existing placed structure still has its old `OwnerCharacterGuid` baked in, so the game no longer recognizes the player as the owner. ## Technical Notes

**Key insights to carry forward:**

1. **The world's procedural content (trees, rocks, ore, enemies, siphon points) is NOT serialized per-instance in the .sav.** It's generated from the world seed at load time and only state deltas are saved (probably under `LVLS` or in a JSON section we haven't categorized). The 15 baseline `SPWN` records in DiffTest A are some kind of system entities — NOT trees. User explicitly clarified this.

2. **The `.sav` is a chunked binary with offset tables.** Adding bytes anywhere reshuffles offsets across the file — naive whole-file diffs are useless. Always work at the chunk level.

3. **Same chest in two saves of the same world differs by only 2 bytes** (tick/hash drift). This is the cleanest possible signal for "the chest data is portable."

4. **`body_first4` and class-ref indices are per-world.** They reference entries in this world's local `CDEF`/`CNIX` tables. Same class in a different world would have different indices. For cross-world detection we need a CNIX-name-based matcher.

5. **Bug #001 root cause is `OwnerCharacterGuid` baked into every placed structure.** Fix path: during world conversion, rewrite every chest/totem/etc.'s `OwnerCharacterGuid` to match the new world owner. Easy fix once we have the SPWN walker.

**No new dependencies added. No config changes.**
