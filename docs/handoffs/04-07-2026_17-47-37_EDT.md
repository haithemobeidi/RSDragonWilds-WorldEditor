## Technical Notes (Key Carry-forward Insights)

1. **Portable cross-world signature recipe.** For any structure class:
   - Capture an EMPTY fresh specimen in a test world
   - Note its `body_length`
   - Read the uint32 at offset 0x4d → that's the `component_count`
   - Verify the 9 bytes at offset 0x44 are `01 0a 02 00 00 f9 03 00 00`
   - Add `(body_length, component_count, display_name)` to `KNOWN_STRUCTURES`
   - Done — works in any world

2. **The 9-byte constant header at 0x44 is universal.** Every SPWN actor record we've seen has it. Trees, system entities, chests — all start their property block with the same prefix. It's likely a UE4 NameProperty/StructProperty type indicator.

3. **`body_first4` is NOT a stable class identifier across worlds.** It's a uint32 index into the world's local CDEF table. The same class appears at different first4 values in different worlds (offset by +0x10 between DiffTest and Gielinor for ash chest).
