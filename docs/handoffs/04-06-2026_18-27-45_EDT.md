## 🎯 The Persistence Breakthrough

### The two bytes that determine world mode

| Byte | Location | Standard | Custom | Notes |
|------|----------|----------|--------|-------|
| **A** | `L_World + 9` (uint32 LE) | `0x00 00 00 00` | `0x03 00 00 00` | UI display cache |
| **B** | First byte of `CustomDifficultySettings` PROP field | `0x00` | `0x03` | **Persistent storage** |

### Why we kept failing earlier

When we set ONLY byte A (the L_World+9 enum):
1. The game would show the world as Custom in the menu temporarily
2. Joining would let you enter the world
3. But gameplay used Standard difficulty values (because byte B was still 0)
4. On save, the game's serializer would overwrite byte A back to 0 (mismatch with byte B)

When we set BOTH bytes:
1. Menu shows Custom ✅
2. Gameplay actually uses custom difficulty values ✅
3. **Saves persist** ✅
4. Survives game restarts ✅

### How to find byte B in any save

The `CustomDifficultySettings` field is field index 8 in the WorldSaveSettings PROP block:

```python
import struct
data = open(sav_path, 'rb').read()
prop_p = data.find(b'PROP')
count = struct.unpack('<I', data[prop_p+8:prop_p+12])[0]   # = 13
data_start = prop_p + 12 + count * 4
offsets = struct.unpack(f'<{count}I', data[prop_p+12 : prop_p+12 + count*4])
cds_pos = data_start + offsets[8]   # ← byte B is at this offset
```

This is what `WorldSave._find_mode_byte_offsets()` does in the parser now.

### Priority 1 — Fix `parser.py` difficulty methods (HIGH)
The editor's difficulty value editing is broken because `_find_difficulty_entries` targets the named TagName/NameProperty entries (which the game doesn't read at runtime) instead of the L_World+17 secondary copy. Rewrite to:
- Find `L_World\0`
- Skip 9 bytes (pad + difficulty enum)
- Skip 4 bytes (version)
- Read N float32 values (where N = number of entries the game allocated, can be inferred from file structure or named entries count)
- Editing should write to the floats here, not to the named entries

Then the World Data tab's difficulty value sliders will actually work in-game.
